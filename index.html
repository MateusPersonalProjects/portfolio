About Me

I am a Computer Science student at the Federal University of Paraná (UFPR) with a strong interest in Software Engineering, Data Engineering, and low-level systems. I am passionate about programming languages, efficient design, and the challenge of building scalable and high-performance solutions. My experience includes practical work with C, C++, Python, JavaScript, and Node.js, as well as knowledge of SQL, PostgreSQL, and Linux, spanning software development, data analysis, and data cleaning. As an undergraduate research fellow at the GDAI Laboratory (UFPR), I developed a system for cleaning and normalizing environmental data, including automatic geographic location discovery for integration into a biological data warehouse. This work led to the publication and presentation of the paper “Desvendando Dados Corrompidos: Uma Jornada de Limpeza, Transformação e Geolocalização em Registros Ambientais” at ERBD 2025, where I also achieved 2nd place in the Academic SQL Marathon. I am currently working on a new project focused on disambiguating geographic locations in text using Knowledge Graphs, leveraging NLP (spaCy), Wikidata, and the Google Geocoding API to enrich spatial databases with greater semantic context and precision. I am seeking opportunities to apply and expand my knowledge in software engineering, data engineering, and systems programming, combining academic research, programming, and data analysis to create impactful technological solutions.

Recent Projects

Street Fighter

StreetFighter is a simple and fun implementation of a Street Fighter game using the C programming language and the Allegro 5 library. The objective of this project was to learn more about event-oriented programming by creating something different.

DBZote

DBZote is a relational SQL database built entirely from scratch in C, inspired by the internal architecture of SQLite. The project is designed as a single-file implementation to make compilation and experimentation easy while exploring the low-level mechanisms of a database engine. At its current stage, DBZote works as an append-only database with a predefined schema, serving as a foundation for future enhancements. The next major milestone is the implementation of B-Trees to enable indexing, faster queries, and more advanced SQL functionality. This project reflects a deep dive into database internals, including storage formats, query execution, and system-level programming.

MallocQueiro

mallocQueiro is a simplified reimplementation of the C malloc function, written entirely in AMD64 (x86-64) Assembly to provide a hands-on understanding of dynamic memory allocation. This project focuses on exposing the fundamental concepts behind how operating systems and the C runtime manage heap memory, including system calls, alignment, and memory blocks. Although not intended for production use, mallocQueiro serves as an educational tool for exploring low-level memory management and systems programming. Future plans include adding support for free, improving memory reuse, and refining the allocator’s internal logic to handle fragmentation more efficiently.

Shopee Finder

Shopee Finder was created with the intention of helping collectors find collectibles more easily on Shopee. Having to look at the website multiple times for new products can be tiring when done for many days, weeks, and months. In order to solve this problem, I created Shopee Finder, an assistant for your searches on Shopee! Shopee Finder was built entirely in Python and uses Selenium, Beautiful Soup to perform web scraping of the site and inform you about new products via email. Additionally, it will generate a CSV file with all the items found in the search that can be accessed directly from the project folder. Feel free to use the project!
